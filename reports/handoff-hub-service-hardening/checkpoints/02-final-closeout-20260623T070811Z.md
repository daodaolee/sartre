## 关键决策

- migration 文件视为 ledger 记录后的不可变历史；对 replay 查询新增 `002` migration，而不是只修改 `001`。
- replay 是 SSE 的离线恢复读模型，不替代在线推送。
- local Dev/QA fixture 是测试/脚本 helper，不是鉴权身份系统。
- clients 继续通过 packages/contracts 和 packages/sdk 使用 Hub API，不 import `apps/hub-api/src/**`。

## 输出约束

- 后续 Web Console 用户页面对接必须消费 SDK/contracts，不得引用 Hub API 内部 repository/application/controller。
- Hub payload 保持 metadata/free-form，不强制 repo、branch、commit range、change report 等岗位字段。
- 若引入 auth/RBAC、queue worker、远程 Codex/Claude 控制，需要另起 OpenSpec change。

## 风险和待办

- 当前 migration runner 可用于本地和测试启动；部署脚本接入需要后续单独 goal。
- replay endpoint 当前仅按 tenant + endpoint + cursor 过滤，未包含 auth 校验；生产鉴权需要后续设计。
- Web Console 操作页尚未接入 replay/commands；建议作为下一 goal。

## 恢复说明

- 若 replay 行为出错，优先检查 `PostgresHandoffRepository.replayEvents()`、`HandoffEventsController.replay()`、`handoffEventReplayResponseSchema` 和 `HandoffHubClient.replayEvents()`。
- 若 migration 幂等出错，优先检查 `migrations.ts`、`migration-files/*.sql` 和本地 `handoff_schema_migrations` ledger。
- 若错误响应漂移，优先检查 `HandoffHttpExceptionFilter` 与 domain error `category`。
- 本 checkpoint 之后的页面对接必须从 regression report 和 acceptance checklist 继续，不应重新定义服务端边界。
