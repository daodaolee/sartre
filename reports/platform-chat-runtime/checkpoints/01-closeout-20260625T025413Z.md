# Platform Chat Runtime Closeout

## Scope

本 checkpoint 收尾 `platform-chat-runtime` goal：落地 provider-neutral Conversation Ledger、Hub API 持久化/HTTP 接口、SDK helper、以及 Web Console 只读会话投影。

## Completed

- 新增 `packages/contracts` conversation schemas/types，覆盖 conversation、message、reference、tool invocation、summary checkpoint、model run、context projection、list/detail response。
- 新增 `packages/sdk` conversation client helpers：create、append message、record tool invocation、create summary/projection、record model run、list/detail。
- Hub API 新增 shared `DatabaseModule`，并新增 `conversation` bounded context、Postgres migration `005_platform_chat_runtime.sql`、repository、application service、HTTP controller。
- Web Console 新增“会话”工作区：按当前 endpoint identity 读取 conversation list/detail，展示 ordered messages、summary checkpoints、context projections 和 audit facts。
- Web Console 本 slice 保持 read-only，没有 composer、streaming、executor 控件。

## Limitations

- No real Codex/Claude/OpenAI execution is called in this goal.
- No streaming token transport or Web Console composer is implemented.
- No provider gateway routing is implemented.
- No semantic memory embedding or recall-before-run is implemented.
- Web Console selects the first visible conversation for the read-only detail projection; multi-conversation selection can be added in the next UI goal.
